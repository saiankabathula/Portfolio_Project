Project Overview: User Management System

Objective: Develop a web-based user management system that includes user sign-up, password recovery, and hint question functionality. The system will use MySQL for database management and Java Servlets for server-side logic.

Features:

  • User Registration (Sign-Up):
    • Form: Users provide details such as username, password, and a hint question with an answer during registration.
    • Validation: Ensure that the username are unique and that the password meets security criteria.
    • Storage: Store user details in the MySQL database, including the hint question with its answer.
  • Password Recovery:
    • Request Password Reset: Users can request a password reset if they forget their password. They need to provide their username only.
    • Hint Question Verification: To ensure identity, the system verifies the user's hint question and answer.
    • Reset Password: After successful verification, allow the user to set a new password. Store the new password securely in the database.
  • Hint Question Management:
    • Set Hint Question: During registration, users can select or change their hint question and provide the answer.
    • Change Hint Question: Users can update their hint question and answer through their profile settings.

Components:

  • Frontend:
    • HTML/CSS/JavaScript: New user registration, login, forgot password, and profile management pages. Include forms for user input and validation scripts.
  • Backend:
    • Java Servlets: Handle HTTP requests for user registration, password recovery, and hint question management. Process form submissions and perform necessary operations.
    • Business Logic: Implement logic for validating user input, interacting with the database, and handling security concerns.
  • Database:
    • MySQL: Stores user information such as ID, username, password, hint question, and hint answer.

Workflow:

  • User Registration: Users complete a registration form with their details and hint question. The servlet processes the form, validates inputs, hashes the password, and stores the data in the users table.
  • Password Recovery: Users request a password reset by providing hint question and prompts the user to answer it. If the answer is correct, the user is allowed to set a new password, which is hashed and updated in the database.
  • Hint Question Management: During registration, users select and answer a hint question. Users can update their hint question and answer from their profile settings. The servlet handles these updates and ensures the new information is securely stored.

Security Considerations:

  • Input Validation: Validate user inputs both on the client side and server side to ensure data integrity.

Technologies Used:

  • Java Servlets: For managing HTTP requests and server-side processing.
  • MySQL: For storing user data and managing authentication details.
  • JDBC: For database connectivity and operations.
  • HTML/CSS/JavaScript: For building the user interface and client-side interactions.
